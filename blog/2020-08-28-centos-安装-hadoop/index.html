centos 安装 hadoop

安装JDK

上传JDK包到centos.

解压

tar xvf server-jre-8u251-linux-x64.tar.gz

配置环境变量

vi /etc/profile

添加Java环境变量:

export JAVA_HOME=/home/jdk1.8.0_251
export PATH=$PATH:$JAVA_HOME/bin

使环境变量生效

source /etc/profile

检查Java安装

java -version

正常输出:

java version "1.8.0_251"
Java(TM) SE Runtime Environment (build 1.8.0_251-b08)
Java HotSpot(TM) 64-Bit Server VM (build 25.251-b08, mixed mode)

安装sshd, pdsh; sshd系统自带, 现在只安装pdsh.需要先安装epel

yum install -y epel-release.noarch
yum install -y pdsh.x86_64

下载最新稳定版hadoop, 当前是2.10.0

wget https://mirrors.tuna.tsinghua.edu.cn/apache/hadoop/common/hadoop-2.10.0/hadoop-2.10.0.tar.gz

解压

tar xvf  hadoop-2.10.0.tar.gz

hadoop有三种部署模式:

  • 单机模式
  • 伪分布式模式
  • 集群模式

我们以伪分布式模式安装.

安装步骤:

下面所有文件修改均针对hadoop目录下文件

编辑etc/hadoop/core-site.xml:

<configuration>
    <property>
        <name>fs.defaultFS</name>
        <value>hdfs://localhost:9000</value>
    </property>
</configuration>

编辑etc/hadoop/hdfs-site.xml:

<configuration>
    <property>
        <name>dfs.replication</name>
        <value>1</value>
    </property>
</configuration>

检查是否设置ssh免密码登录

ssh localhost

如果不能免密登录, 设置ssh免密登录:

ssh-keygen -t rsa -P '' -f ~/.ssh/id_rsa
cat ~/.ssh/id_rsa.pub >> ~/.ssh/authorized_keys
chmod 0600 ~/.ssh/authorized_keys

启动服务

  1. 格式化文件系统
bin/hdfs namenode -format
  1. 启动 NameNode 服务 和 DataNode 服务:
sbin/start-dfs.sh

发现启动报错

localhost: Error: JAVA_HOME is not set and could not be found.
localhost: Error: JAVA_HOME is not set and could not be found.
Starting secondary namenodes [0.0.0.0]
The authenticity of host '0.0.0.0 (0.0.0.0)' can't be established.
ECDSA key fingerprint is SHA256:v62FRQvEyLKc3YJSY50MixeSqmRiuRt0KqpoOArEOkY.
ECDSA key fingerprint is MD5:cf:74:7d:e2:66:17:8d:29:7d:5e:41:0c:25:94:88:7a.
Are you sure you want to continue connecting (yes/no)? yes
0.0.0.0: Warning: Permanently added '0.0.0.0' (ECDSA) to the list of known hosts.
0.0.0.0: Error: JAVA_HOME is not set and could not be found.

应该是hadoop进程ssh到主机后session无JAVA_HOME环境变量, /etc/profile配置未起作用, 在~.bashrc下引用/etc/profile使其生效.

bash有“交互”和“登陆”两种属性，而执行“profile”系列还是“rc”系列，就与shell的这两个属性有关。原理上讲，“登陆shell”启动时会加载“profile”系列的startup文件，而“交互式非登陆shell”启动时会加载“rc”系列的startup文件。

执行:

 vi /root/.bashrc

添加:

source /etc/profile
  1. 重新启动 NameNode 服务 和 DataNode 服务:
sbin/start-dfs.sh

控制台输出

Starting namenodes on [localhost]
localhost: starting namenode, logging to /home/hadoop-2.10.0/logs/hadoop-root-namenode-localhost.localdomain.out
localhost: starting datanode, logging to /home/hadoop-2.10.0/logs/hadoop-root-datanode-localhost.localdomain.out
Starting secondary namenodes [0.0.0.0]
0.0.0.0: starting secondarynamenode, logging to /home/hadoop-2.10.0/logs/hadoop-root-secondarynamenode-localhost.localdomain.out

环境变量配置成功, 服务启动成功了.

有些情况下datanode会启动失败,这时候需要根据日志确定数据目录,清空后格式化数据目录,重新启动
