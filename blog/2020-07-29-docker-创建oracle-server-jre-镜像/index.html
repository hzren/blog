docker 创建Oracle Server Jre 镜像

先上Dockerfile配置:

FROM centos:7
# 维护者 
MAINTAINER your-name
# 将jdk压缩包添加到容器的 /root 目录，解压后目录名称为jdk1.8.0_211
#ADD server-jre-8u251-linux-x64.tar.gz /root
COPY jre /jre
# 配置JAVA_HOME环境变量
ENV JAVA_HOME /jre/
# 将JAVA_HOME/bin 添加至PATH环境变量
ENV PATH $JAVA_HOME/bin:$PATH
# 启动容器执行的命令，仅用于验证安装配置是否正确，生产环境使用需注释后再build
#CMD java -version
#ENTRYPOINT ["/jre/bin/java", "-version"]

步骤:

  1. 下载oracle server jre, 我当前下载下来是8u251版本
  2. 上传到centos, Dockfile同层目录下
  3. 解压缩, 修改解压缩后文件夹名字为jre
  4. 在当前目录执行docker build: docker build -t jdk:jre8 .
  5. 执行完成后, 镜像被保存到本地仓库, jdk 为 image名称, jre为版本, -t 代表 tag

遇到的坑:

  1. 不能FROM scratch, 继承自空白镜像连sh命令都没有, Dockerfile 中的CMD和ENTRYPOINT命令是以sh -c xxxxx 执行的, 没sh命令根本执行不了
  2. 不能FROM busybox:latest , 也是一样找不到sh命令, docker本身是去找bash, 但是busybox 不包含bash
  3. 不能FROM alpine:latest, 理由同上, 也是找不到bash

最后FROM centos:7 好了

docker 镜像删除命令

docker 删除所有无用镜像, 容器 :

docker system prune

docker 删除无用镜像:

docker rmi docker images | grep "<none>" | awk {'print $3'}
